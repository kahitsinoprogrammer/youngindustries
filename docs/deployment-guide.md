# Deployment Guide

This guide covers the deployment of the Young Industries customized Snipe-IT project from this repository.

> [!IMPORTANT]
> The checked-in `docker-compose.yml` uses the upstream image `snipe/snipe-it:${APP_VERSION:-latest}`. If you need to deploy the custom code in this repository exactly as-is, use the source-based installation steps below or update the Compose setup to build from this repository instead of pulling the upstream image.

## 2. Installation Steps

### 2.1 Source-based server deployment

1. Clone the repository.

```bash
git clone https://github.com/kahitsinoprogrammer/youngindustries.git
cd youngindustries
```

2. Install backend dependencies.

```bash
composer install --no-dev --optimize-autoloader
```

3. Install frontend dependencies and build production assets.

```bash
npm install
npm run production
```

4. Create the environment file and generate the Laravel application key.

```bash
cp .env.example .env
php artisan key:generate
```

5. Update `.env` with the correct application, database, mail, and any integration-specific settings for your environment.

6. Run the database migrations.

```bash
php artisan migrate --force
```

7. Prepare caches and the storage symlink.

```bash
php artisan config:cache
php artisan route:cache
php artisan view:cache
php artisan storage:link
```

8. Configure the web server to point to the `public/` directory, not the repository root.

9. Ensure the web server user can write to `storage/` and `bootstrap/cache/`.

10. Start or reload the web server and verify the application loads successfully in the browser.

### 2.2 Web server notes

| Server | Required setup |
| --- | --- |
| Apache | Enable `mod_rewrite`, allow Laravel `.htaccess`, and point the virtual host document root to `public/` |
| Nginx | Point the server root to `public/` and route requests through Laravel's `index.php` |

### 2.3 Optional container path

| File | Intended use | Note |
| --- | --- | --- |
| `docker-compose.yml` | Production-style container stack | Pulls the upstream `snipe/snipe-it` image rather than building this fork |
| `dev.docker-compose.yml` | Local development stack | Builds locally with `Dockerfile.alpine` and uses `.env.dev.docker` |
| `.env.docker` | Container environment template | Includes `APP_VERSION`, `APP_PORT`, and database container settings |

```text

OLLAMA_API_KEY=your_api_key
OLLAMA_AUTH_HEADER=Authorization
OLLAMA_AUTH_SCHEME=Bearer
OLLAMA_MODEL=your-cloud-enabled-model
```

## 4. Post-deployment checks

| Check | Expected result |
| --- | --- |
| Application URL loads | Login page or setup-complete home page appears |
| Database connection works | No DB connection errors in the UI or logs |
| File upload path works | Uploads and storage symlink behave correctly |
| Mail configuration works | Test emails send successfully |
| Optional integrations work | MongoDB, Firebase, Jira, S3, LDAP, SAML, or Ollama features only if enabled |
